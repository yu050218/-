# 局域网访问排查清单

## 服务器端检查（运行服务器的电脑）

### 1. 确认服务器正在运行
```bash
netstat -ano | findstr ":8000"
```
应该显示：`TCP    0.0.0.0:8000           0.0.0.0:0              LISTENING`

### 2. 确认防火墙规则
```bash
netsh advfirewall firewall show rule name="Allow Port 8000 HTTP"
```
应该显示：`已启用: 是`, `操作: 允许`

### 3. 测试本地访问
```bash
curl http://192.168.1.110:8000
```

## 其他设备检查（手机/平板/其他电脑）

### 1. 测试端口连通性

**Windows 设备：**
```powershell
Test-NetConnection -ComputerName 192.168.1.110 -Port 8000
```

**Android 设备：**
安装 "Network Info II" 或 "Ping" 应用，测试能否连接到 192.168.1.110:8000

**iOS 设备：**
使用 "iNetTools" 或 "Ping" 应用测试

### 2. 尝试 HTTP 请求
```bash
curl http://192.168.1.110:8000
```
或者在浏览器直接访问 http://192.168.1.110:8000

### 3. 检查浏览器控制台
- 打开浏览器开发者工具（F12）
- 查看 Console 和 Network 标签页
- 记录任何错误信息

## 常见问题解决方案

### 问题1：路由器 AP 隔离
**症状：**ping 通但无法访问 HTTP
**解决：**登录路由器管理页面（通常是 192.168.1.1 或 192.168.0.1），关闭 "AP隔离" 或 "客户端隔离" 功能

### 问题2：防病毒软件拦截
**解决：**临时关闭 360安全卫士、腾讯电脑管家等软件的防火墙功能

### 问题3：网络位置被识别为公共网络
**解决：**确保连接的网络被设置为 "专用网络" 或 "工作网络"

### 问题4：代理设置
**解决：**确保设备没有设置代理，直接连接网络

### 问题5：任何措施后同一局域网内设备仍然不能访问网址
**解决：**在管理员权限下的命令提示符中暂时关闭防火墙
```bash
netsh advfirewall set allprofiles state off  
```
